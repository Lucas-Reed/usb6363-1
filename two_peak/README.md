# 双峰锁定重写说明

这个目录是双峰锁定的新实现，目标是避免旧版再次变成一个巨大文件。

## 边界

- `config.py`：集中保存实验参数，后续 WebUI 尽量从这里读取默认值。
- `signal.py`：只做信号处理和寻峰，不控制硬件。
- `pi.py`：只做 PI 计算，不控制硬件。
- `lock_engine.py`：把一帧 AI 数据变成峰值、误差和 AO 建议值。

## 暂时不做的事

自动双峰识别先不照搬旧程序。这个算法需要结合真实波形重新讨论，否则很容易再次越改越复杂。

## 硬件规则

本目录内的文件不允许导入 `nidaqmx`。需要采集或输出时，只能通过 `usb6363_client.py` 调用底层 API 服务。
